# NanoByte

NanoByte is a connected health application concept built around an ingestible nanobot that runs on stomach fluid. Once consumed, the nanobot communicates with a paired mobile application to surface real-time personal health diagnostics in a digestible, easy-to-understand format.

## What It Does

NanoByte gives users a live view of their internal health data, including:

- Vitamin and nutrient levels
- Blood sugar
- Cholesterol
- Personalized health recommendations based on your diagnostics

No lab visits. No waiting for results. Just your health data, surfaced in real time.

## Origin

NanoByte was originally conceived as a group project during my graduate program at Temple University. The initial prototype was designed in Figma and served as a static proof of concept.

This version is a functional rebuild of that original idea, developed independently using Google Stitch and Claude Code.

## How It Was Built

The prototype was built using a two-step workflow:

1. **Google Stitch** was used to design and wireframe the user interface quickly and intuitively
2. The exported code was then brought into **Claude Code**, which transformed the wireframe into a fully functional application

This workflow allowed a non-engineer to go from concept to working prototype in a fraction of the time the original Figma version required.

## Purpose

This project was built to demonstrate how modern AI-assisted tools are changing what product managers can build independently. It is intended as a portfolio piece showcasing product thinking, rapid prototyping, and UI design.

## Status

Prototype. This is a concept application and not connected to real medical hardware or data.
